## Endpoint

```
POST /users
```

Registers a new user in the system. Supports creating users with different roles (admin, technician, or client) and automatically hashes the password using bcrypt before storage.

## Request Body

<ParamField body="name" type="string" required>
  Full name of the user
  
  **Example**: `"Santiago Portillo"`
</ParamField>

<ParamField body="email" type="string" required>
  Email address (must be unique)
  
  **Example**: `"santiagoportillo@gmail.com"`
  
  <Note>Email addresses must be unique across all users in the system</Note>
</ParamField>

<ParamField body="password" type="string" required>
  User password (minimum 6 characters)
  
  **Example**: `"123456"`
  
  <Note>Passwords are automatically hashed using bcrypt with a generated salt</Note>
</ParamField>

<ParamField body="role" type="enum" required>
  User role: `admin`, `technician`, or `client`
  
  **Default**: `client`
  
  **Options**:
  - `admin` - Full system access
  - `technician` - Support staff role
  - `client` - Standard user role
</ParamField>

<ParamField body="company" type="string">
  Company name (only for client role)
  
  **Example**: `"Acme Corporation"`
  
  This field is optional and only used when creating a client user.
</ParamField>

<ParamField body="specialty" type="string">
  Technical specialty (only for technician role)
  
  **Example**: `"Network Infrastructure"`
  
  This field is optional and only used when creating a technician user.
</ParamField>

## Response

<ResponseField name="id" type="string">
  UUID of the created user
</ResponseField>

<ResponseField name="name" type="string">
  Full name of the user
</ResponseField>

<ResponseField name="email" type="string">
  Email address
</ResponseField>

<ResponseField name="role" type="string">
  User role (admin, technician, or client)
</ResponseField>

<ResponseField name="createdAt" type="datetime">
  Timestamp when the user was created
</ResponseField>

<ResponseField name="updatedAt" type="datetime">
  Timestamp when the user was last updated
</ResponseField>

<Note>
  The `password` field is never returned in the response for security reasons
</Note>

## Password Hashing Implementation

Passwords are hashed using bcrypt before storage:

```typescript
const salt = await bcrypt.genSalt();
const hashedPassword = await bcrypt.hash(createUserDto.password, salt);
```

The password field in the User entity has `select: false` configured, ensuring it's never accidentally exposed in queries.

## Example Request

<CodeGroup>

```bash cURL
curl -X POST http://localhost:3000/users \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Santiago Portillo",
    "email": "santiagoportillo@gmail.com",
    "password": "123456",
    "role": "client",
    "company": "Tech Solutions Inc"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3000/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Santiago Portillo',
    email: 'santiagoportillo@gmail.com',
    password: '123456',
    role: 'client',
    company: 'Tech Solutions Inc'
  })
});

const user = await response.json();
```

```python Python
import requests

response = requests.post(
    'http://localhost:3000/users',
    json={
        'name': 'Santiago Portillo',
        'email': 'santiagoportillo@gmail.com',
        'password': '123456',
        'role': 'client',
        'company': 'Tech Solutions Inc'
    }
)

user = response.json()
```

## Response Codes

<ResponseField name="201" type="Created">
  User successfully created
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid input data:
  - Email already exists
  - Missing required fields
  - Password shorter than 6 characters
  - Invalid email format
</ResponseField>
