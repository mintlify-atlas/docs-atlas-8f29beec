## Description

Returns the profile information of the currently authenticated user. This endpoint requires a valid JWT token in the Authorization header.

The response contains the user data extracted from the JWT token payload, including user ID, email, and role.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from the `/auth/login` endpoint.
  
  Format: `Bearer <access_token>`
</ParamField>

## Response

<ResponseField name="userId" type="string">
  The user's unique identifier (UUID format). Extracted from the JWT token's `sub` claim.
</ResponseField>

<ResponseField name="email" type="string">
  The user's email address.
</ResponseField>

<ResponseField name="role" type="string">
  The user's role in the system. Possible values:
  - `admin` - Administrator with full access
  - `technician` - Technical staff member
  - `client` - Regular client user
</ResponseField>

## Error Responses

<ResponseField name="401 Unauthorized" type="object">
  Returned when the JWT token is missing, invalid, expired, or malformed.
  
  ```json
  {
    "statusCode": 401,
    "message": "Unauthorized"
  }
  ```
</ResponseField>

## Implementation Details

Implemented in:
- Controller: `src/modules/auth/auth.controller.ts:31`
- JWT Strategy: `src/modules/auth/jwt.strategy.ts:19`

The authentication flow:
1. Extracts JWT token from Authorization header (Bearer scheme)
2. Validates token signature using the secret key
3. Checks token expiration
4. Executes the `validate()` method in JwtStrategy which transforms the payload
5. Injects the user object into `request.user`
6. Returns the user profile data

## Usage in Protected Routes

This endpoint demonstrates how authentication works across the API. Any endpoint decorated with `@UseGuards(AuthGuard('jwt'))` will require the same Bearer token authentication and will have access to the user profile via `request.user`.
