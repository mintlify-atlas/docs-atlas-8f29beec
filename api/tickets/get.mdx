## Endpoint

```
GET /tickets/:id
```

Retrieves the complete details of a specific ticket by its unique identifier.

## Authentication

<Note>
This endpoint requires JWT Bearer authentication.
</Note>

## Path Parameters

<ParamField path="id" type="string" required>
  Unique identifier of the ticket to retrieve
  
  **Format:** UUID v4
  
  **Example:** `123e4567-e89b-12d3-a456-426614174000`
</ParamField>

## Response

Returns a single Ticket object with complete details.

<ResponseField name="id" type="string">
  Unique identifier (UUID) of the ticket
</ResponseField>

<ResponseField name="title" type="string">
  Title of the ticket
</ResponseField>

<ResponseField name="description" type="string">
  Detailed description of the issue
</ResponseField>

<ResponseField name="status" type="string">
  Current status: `open`, `in_progress`, `resolved`, or `closed`
</ResponseField>

<ResponseField name="priority" type="string">
  Priority level: `low`, `medium`, or `high`
</ResponseField>

<ResponseField name="client" type="object">
  Client who created the ticket
  
  <Expandable title="Client Object">
    <ResponseField name="id" type="string">
      Client's unique identifier
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Client's name
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Client's email address
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="technician" type="object">
  Assigned technician (null if unassigned)
  
  <Expandable title="Technician Object">
    <ResponseField name="id" type="string">
      Technician's unique identifier
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Technician's name
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Technician's email address
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="category" type="object">
  Category the ticket belongs to
  
  <Expandable title="Category Object">
    <ResponseField name="id" type="string">
      Category's unique identifier
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Category name
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string">
  Timestamp when the ticket was created (ISO 8601 format)
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Timestamp when the ticket was last updated (ISO 8601 format)
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X GET https://api.example.com/tickets/123e4567-e89b-12d3-a456-426614174000 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://api.example.com/tickets/123e4567-e89b-12d3-a456-426614174000',
  {
    headers: {
      'Authorization': 'Bearer YOUR_JWT_TOKEN'
    }
  }
);

const ticket = await response.json();
```

```python Python
import requests

ticket_id = '123e4567-e89b-12d3-a456-426614174000'
url = f'https://api.example.com/tickets/{ticket_id}'
headers = {'Authorization': 'Bearer YOUR_JWT_TOKEN'}

response = requests.get(url, headers=headers)
ticket = response.json()
```

</CodeGroup>

## UUID Validation

<Warning>
The ticket ID must be a valid UUID v4 format. Invalid UUIDs will return a 400 Bad Request error.
</Warning>

**Valid UUID format:**
```
123e4567-e89b-12d3-a456-426614174000
```

**Invalid formats:**
- `123` (too short)
- `not-a-uuid` (invalid format)
- `123e4567e89b12d3a456426614174000` (missing hyphens)
