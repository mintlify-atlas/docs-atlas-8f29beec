## Overview

The TechHelpDesk API provides powerful filtering capabilities to help you find specific tickets. You can filter by status, priority, assigned technician, or client, and combine multiple filters for precise queries.

## Available Filters

The `/tickets` endpoint supports four filter parameters:

<ParamField query="status" type="string">
  Filter by ticket status: `open`, `in_progress`, `resolved`, or `closed`
</ParamField>

<ParamField query="priority" type="string">
  Filter by priority level: `low`, `medium`, or `high`
</ParamField>

<ParamField query="technicianId" type="string">
  Filter by assigned technician's UUID
</ParamField>

<ParamField query="clientId" type="string">
  Filter by client's UUID who created the ticket
</ParamField>

<Info>
  All filters are optional. Omitting all filters returns all tickets (subject to role-based access control).
</Info>

### Filter Logic (from tickets.service.ts:33-46)

```typescript
async findAll(params: FindAllTicketsParams) {
  const { status, priority, technicianId, clientId } = params;
  const where: FindOptionsWhere<Ticket> = {};

  if (status) where.status = status;
  if (priority) where.priority = priority as any;
  if (technicianId) where.technician = { id: technicianId };
  if (clientId) where.client = { id: clientId };

  return this.ticketRepository.find({
    where,
    relations: ['client', 'technician', 'category'],
    order: { createdAt: 'DESC' },
  });
}
```

<Note>
  Results are always sorted by creation date in descending order (newest first).
</Note>

## Filter by Status

Filter tickets by their current lifecycle state.

### Status Values

- `open` - New tickets awaiting assignment
- `in_progress` - Tickets actively being worked on
- `resolved` - Tickets marked as resolved
- `closed` - Permanently closed tickets

### Example Requests

<CodeGroup>

```bash All Open Tickets
curl -X GET "https://api.techhelpdesk.com/tickets?status=open" \
  -H "Authorization: Bearer <token>"
```

```bash All Resolved Tickets
curl -X GET "https://api.techhelpdesk.com/tickets?status=resolved" \
  -H "Authorization: Bearer <token>"
```

```bash Tickets In Progress
curl -X GET "https://api.techhelpdesk.com/tickets?status=in_progress" \
  -H "Authorization: Bearer <token>"
```

```javascript JavaScript
const response = await fetch(
  'https://api.techhelpdesk.com/tickets?status=open',
  {
    headers: {
      'Authorization': `Bearer ${token}`
    }
  }
);

const openTickets = await response.json();
console.log(`Found ${openTickets.length} open tickets`);

```

## Combining Multiple Filters

You can combine any filters to create precise queries.

### Example Combinations

<Tabs>
  <Tab title="High Priority + Open">
    Find urgent tickets that need assignment:
    
    ```bash
    curl -X GET "https://api.techhelpdesk.com/tickets?priority=high&status=open" \
      -H "Authorization: Bearer <token>"
    ```
    
    **Use case:** Identify critical unassigned tickets
  </Tab>
  
  <Tab title="Technician + Status">
    Check a technician's active workload:
    
    ```bash
    curl -X GET "https://api.techhelpdesk.com/tickets?technicianId=tech-001&status=in_progress" \
      -H "Authorization: Bearer <token>"
    ```
    
    **Use case:** Monitor technician capacity (max 5 in_progress tickets)
  </Tab>
  
  <Tab title="Client + Priority">
    View a client's high-priority tickets:
    
    ```bash
    curl -X GET "https://api.techhelpdesk.com/tickets?clientId=client-001&priority=high" \
      -H "Authorization: Bearer <token>"
    ```
    
    **Use case:** Client dashboard showing urgent issues
  </Tab>
  
  <Tab title="All Filters">
    Very specific query:
    
    ```bash
    curl -X GET "https://api.techhelpdesk.com/tickets?status=in_progress&priority=high&technicianId=tech-001&clientId=client-001" \
      -H "Authorization: Bearer <token>"
    ```
    
    **Use case:** Check if a specific technician is working on a client's urgent ticket
  </Tab>
</Tabs>

### JavaScript Example: Multiple Filters

```javascript
const filters = {
  status: 'open',
  priority: 'high'
};

// Build query string
const queryString = new URLSearchParams(filters).toString();

const response = await fetch(
  `https://api.techhelpdesk.com/tickets?${queryString}`,
  {
    headers: { 'Authorization': `Bearer ${token}` }
  }
);

const tickets = await response.json();
console.log(`Found ${tickets.length} high-priority open tickets`);
```

### Python Example: Dynamic Filters

```python
import requests

def get_filtered_tickets(token, **filters):
    """Fetch tickets with dynamic filters"""
    response = requests.get(
        'https://api.techhelpdesk.com/tickets',
        params=filters,
        headers={'Authorization': f'Bearer {token}'}
    )
    return response.json()

# Example usage
tickets = get_filtered_tickets(
    token=my_token,
    status='open',
    priority='high'
)

print(f"Found {len(tickets)} tickets")
```

## Error Handling

<Accordion title="Invalid Status Value">
  Using an invalid status value returns an empty array, not an error:
  
  ```bash
  GET /tickets?status=invalid
  # Returns: []
  ```
  
  <Note>
    The API doesn't validate enum values in query parameters. Invalid values simply don't match any records.
  </Note>
</Accordion>

<Accordion title="Invalid UUID Format">
  Malformed UUIDs for `technicianId` or `clientId` return empty results:
  
  ```bash
  GET /tickets?technicianId=not-a-uuid
  # Returns: []
  ```
</Accordion>

<Accordion title="Non-existent ID">
  Valid UUID format but non-existent ID returns empty array:
  
  ```bash
  GET /tickets?clientId=00000000-0000-0000-0000-000000000000
  # Returns: []
  ```
</Accordion>

## Implementation Details

The filter implementation uses TypeORM's `FindOptionsWhere` to build dynamic queries.

### Query Building (from tickets.service.ts:33-40)

```typescript
const { status, priority, technicianId, clientId } = params;
const where: FindOptionsWhere<Ticket> = {};

if (status) where.status = status;
if (priority) where.priority = priority as any;
if (technicianId) where.technician = { id: technicianId };
if (clientId) where.client = { id: clientId };
```

<Info>
  Only non-null/undefined filter parameters are added to the WHERE clause. This allows flexible combination of filters.
</Info>

### Generated SQL Example

For a query with status and priority filters:

```sql
SELECT * FROM tickets
LEFT JOIN clients ON tickets.client_id = clients.id
LEFT JOIN technicians ON tickets.technician_id = technicians.id  
LEFT JOIN categories ON tickets.category_id = categories.id
WHERE tickets.status = 'open' 
  AND tickets.priority = 'high'
ORDER BY tickets.createdAt DESC;
```
