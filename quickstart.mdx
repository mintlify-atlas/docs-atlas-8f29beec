# Quickstart Guide

Get TechHelpDesk API up and running quickly with this streamlined guide. For detailed configuration options, see the [Installation Guide](/installation).

## Prerequisites

Before you begin, ensure you have:

- **Node.js** 16+ installed
- **PostgreSQL** 12+ installed or Docker for containerized setup
- **Git** for cloning the repository

## Installation Steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/SantiagoPortillo1/Prueba-de-Desempe-o-M-dulo-Node.js-con-NestJS-y-TypeORM.git
    cd Prueba-de-Desempe-o-M-dulo-Node.js-con-NestJS-y-TypeORM
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
    
    This will install all required packages including NestJS, TypeORM, and PostgreSQL drivers.
  </Step>

  <Step title="Start PostgreSQL database">
    Using Docker Compose (recommended):
    
    ```bash
    docker-compose up -d
    ```
    
    This starts PostgreSQL with the following credentials:
    - **Host**: localhost:5432
    - **Database**: helpdesk_db
    - **User**: admin
    - **Password**: root
    
    <Note>
      If you're using a local PostgreSQL installation, create a database named `helpdesk_db` and update the credentials in your `.env` file.
    </Note>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root:
    
    ```bash
    cp .env.example .env
    ```
    
    Update the `.env` file with your database configuration:
    
    ```bash .env
    # Database Configuration
    DB_HOST=localhost
    DB_PORT=5432
    DB_USERNAME=admin
    DB_PASSWORD=root
    DB_DATABASE=helpdesk_db
    
    # JWT Configuration
    JWT_SECRET=your-secret-key-change-this-in-production
    JWT_EXPIRES_IN=1h
    
    # Server Configuration
    PORT=3000
    NODE_ENV=development
    ```
    
    <Warning>
      Change the `JWT_SECRET` to a secure random string in production environments.
    </Warning>
  </Step>

  <Step title="Run database migrations">
    ```bash
    npm run migration:run
    ```
    
    This creates all necessary database tables and relationships.
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run start:dev
    ```
    
    The API server will start on `http://localhost:3000`
    
    You should see output similar to:
    ```
    [Nest] Application is running on: http://localhost:3000/docs
    ```
  </Step>
</Steps>

## Verify Installation

### Access Swagger Documentation

Open your browser and navigate to:

```
http://localhost:3000/docs
```

You'll see the interactive Swagger UI with all available endpoints documented.

### Make Your First API Call

<Tabs>
  <Tab title="Using cURL">
    Test the API by logging in:
    
    ```bash
    curl -X POST http://localhost:3000/auth/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "admin@example.com",
        "password": "password123"
      }'
    ```
    
    Expected response:
    ```json
    {
      "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "id": "uuid-here",
        "email": "admin@example.com",
        "role": "admin"
      }
    }
    ```
  </Tab>
  
  <Tab title="Using Swagger UI">
    1. Click on **POST /auth/login** endpoint
    2. Click **Try it out**
    3. Enter the request body:
       ```json
       {
         "email": "admin@example.com",
         "password": "password123"
       }
       ```
    4. Click **Execute**
    5. Copy the `access_token` from the response
    6. Click the **Authorize** button at the top of the page
    7. Enter `Bearer YOUR_TOKEN` and click **Authorize**
    
    Now you can test all protected endpoints!
  </Tab>
  
  <Tab title="Using JavaScript">
    ```javascript
    const response = await fetch('http://localhost:3000/auth/login', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        email: 'admin@example.com',
        password: 'password123'
      })
    });
    
    const data = await response.json();
    console.log('Token:', data.access_token);
    ```
  </Tab>
</Tabs>

## Create Your First Ticket

Once authenticated, create a support ticket:

```bash
curl -X POST http://localhost:3000/tickets \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Server not responding",
    "description": "The application server is not responding to requests",
    "clientId": "client-uuid",
    "categoryId": "category-uuid",
    "priority": "high"
  }'
```

<Note>
  Replace `YOUR_TOKEN`, `client-uuid`, and `category-uuid` with actual values. You can get UUIDs from the database or create users and categories first.
</Note>

## Common Issues

<Accordion title="Connection to database refused">
  **Problem**: Cannot connect to PostgreSQL database.
  
  **Solutions**:
  - Ensure PostgreSQL is running: `docker-compose ps` or `sudo systemctl status postgresql`
  - Check database credentials in `.env` file match your PostgreSQL setup
  - Verify the port 5432 is not already in use: `lsof -i :5432`
  - Check PostgreSQL logs for connection errors
</Accordion>

<Accordion title="Migration error: relation already exists">
  **Problem**: Database tables already exist from a previous installation.
  
  **Solutions**:
  - Drop the database and recreate it: `npm run schema:drop && npm run migration:run`
  - Or manually drop tables and re-run migrations
  - Restore from the SQL dump: `psql -U admin -d helpdesk_db < database/dump.sql`
</Accordion>

<Accordion title="JWT token validation fails">
  **Problem**: Protected endpoints return 401 Unauthorized.
  
  **Solutions**:
  - Ensure you're including the Bearer prefix: `Authorization: Bearer YOUR_TOKEN`
  - Check that the JWT_SECRET in `.env` matches the one used to generate the token
  - Verify the token hasn't expired (default: 1 hour)
  - Get a fresh token by logging in again
</Accordion>

<Accordion title="Port 3000 already in use">
  **Problem**: Another application is using port 3000.
  
  **Solutions**:
  - Change the port in `.env`: `PORT=3001`
  - Or stop the application using port 3000: `lsof -ti:3000 | xargs kill -9`
</Accordion>
